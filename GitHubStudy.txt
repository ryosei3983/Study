０：初めにaa

	今回、学習するのは「GitHub」です。
	わかりやすいし便利だからやっておきましょう

	以下、御託

	学ぶ理由はいたって簡単、データのやり取りを簡単にするためです。
	今まで私たちはチーム制作などでデータのやり取りを行う際
	USBの交換やLINEでのデータの受け渡しを行ってきました。
	しかし、それはあまりにも非効率的だと思いませんか？

	そこでGitHubです。	

	ネット環境さえあれば各々が最新のデータをまとめて更新し、
	共有しあえるとても優秀なソフトです。
	これさえあればチーム制作はもっとスムーズに進むことでしょう。
	＊チーム制作でGitHubを使う場合GitHubを扱えない人とは使うことができません

	まあ、一緒にチーム制作をする仲間がいなくてもファイルのバックアップを取る
	ソフトとしても優秀ですので頑張っていきましょう。
	
	
	　
	

１：導入

	GitHubアカウント

	　ブラウザでhttps://github.com/にアクセスして
	　GitHubのアカウントを作ります。

	GitHubDesktop

	　ブラウザでhttps://desktop.github.com/にアクセスして
	　GitHubDesktopをダウンロードします。

	　ダウンロードしたファイルからGitHubDesktopをインストールします。
	  インストールを終えるとGitHubのアカウントの認証が求められます。
	　＊すでにサインイン済の場合求められないかもしれません

２：動かしてみよう（バックアップ編）

	認証を終えたら Create a New Repository on your hard drive... と
	書かれたボタンをクリックし、デスクトップ上にフォルダを作りましょう
	　補足：リポジトリ、フォルダ、ディレクトリ（厳密には違うがだいたい一緒）

	　
	作成したフォルダに適当な中身を書き込んだ.txtファイルを入れてください

	そのあとGitHubDesktopの左下にあるボタンの Commit to master を
	クリックしてください。
	　＊押せない場合はボタン上のTextBoxに適当に入力すれば押せるようになります。
	　＊ボタンが見つからない人は左上の項目が History になっているかもしれません
	　　Changes にクリックで変更すれば見つかるはずです。

	クリックしたら左上にあるボタンの History をクリックしてください。
	すると右画面に大きく.txtファイルに書き込んだ内容が表示されます。
	このボタンを押す行為を通称「コミットする」といいます

	先ほどフォルダに配置した.txtファイルの内容を変更して
	同じようにコミットしてください。

	するとHistoryのほうに先ほどの.txtファイルの内容と
	現在の.txtファイルの内容が見れるはずです。

	こうすることによりファイルやデータ量を少なくしながら
	データのバックアップをとることができます。

	　補足：GitHubDesktopは.txtファイルだけでなくあらゆるデータを
		管理することが可能です。画像ファイルの差分や
		ソースコードの.cppファイルのバックアップも取れます。

		しかしファイルを複数の種類入れると手動で
	　	バックアップを取るファイルを選ぶのが面倒なので
		気をつけましょう。

３：動かしてみよう（チーム制作編）

	先にバックアップ編を見ておくことをお勧めします。

　サーバー上にフォルダ（リモートリポジトリ）を作る

	ブラウザでhttps://desktop.github.com/にアクセスし、
	アカウントでGitHubにサインインした後
	画面右上にある ＋ ボタンから NewRepositry をクリックします

	ファルダに好きな名前を付けましょう
	Repositry name　に記入

	Description にはフォルダの使用用途を書きましょう
	＊必須ではない

	PublicとPrivateがありますが違いは簡単です。
	Publicは他のユーザーから閲覧されます（全世界、７０億人）
	Privateは自分と許可をしたユーザー以外は閲覧できません

	入力し終えたら Create repositry ボタンをクリックで
	サーバー上にフォルダ（リモートリポジトリ）を作成します。

　デスクトップ上にリモートリポジトリをコピーする	

	ここでGitHubDesktopに戻ります。

	すでにDesktop上にGitHubDesktopで作成したフォルダがある場合	

	　左上の Current repository をクリックし、Add をクリックしてください
	　そのあと GitHub.com という項目が出てくるのでそれをクリックして
	　GitHubにサインインを求められたら素直にサインインしてください。

	　するとYour repositories という項目欄に先ほど作ったフォルダが
	　あるはずなのでそれを選択した後、Local Pathに Desktopを
	　選択して Clone ボタンをクリックします。

	　デスクトップ上に先ほどサーバー上に制作したフォルダが
	　コピーされます。

	　このボタンを押す行為を「クローンを作成する」といいます

	Desktop上にGitHubDesktopで作成したフォルダがない場合

	　Clone a repository from the Internet...という項目をクリックし
	　あとはだいたい上と一緒です。

　書き換えた内容をサーバーにアップ（フェッチ）する


	上で作成したクローンのフォルダに適当な.txtファイルを入れます。
	画面上中央当たりに Fetch origin という項目をクリックすると
	ブラウザ上にその.txtファイルがあるのがわかります。

	この行為を「フェッチする」といいます
